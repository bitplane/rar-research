# rars-generated RAR 1.5 Writer Oracles

These archives were generated by `rars` from two tiny deterministic files:

- `payload.txt`: `rar15 oracle payload\n`
- `second.txt`: `rar15 oracle second file\n`

They are focused writer-output fixtures, not vintage WinRAR-authored inputs.
They exist to pin the RAR 1.5 bytes emitted by `rars` after validation with a
historical public reader.

Validation performed with WinRAR/UnRAR 4.20 under Wine:

- `stored.rar`: `unrar t` reports all OK.
- `compressed.rar`: `unrar t` reports all OK.
- `solid.rar`: `unrar t` reports all OK.
- `encrypted.rar`: `unrar t -ppass` reports all OK.
- `comments.rar`: `unrar t` reports all OK and prints archive comment
  `oracle-note`.
- `split-store.rar` + `.r00` + `.r01`: `unrar t` reports all OK.
- `split-encrypted.rar` + `.r00` + `.r01`: `unrar t -ppass` reports all OK.

The split encrypted fixture pins the historical RAR 1.5 behavior that
`FHD_PASSWORD` appears on every split file block, but the CRYPT_RAR15 stream is
continuous across the logical packed member before splitting.
