# RAR 2.9 / 3.x PPMd-Mode Fixtures

Generated by `scripts/generate-ppmd-fixture.py` using RAR 3.00 from
`_refs/wineprefixes/winrar300/`.

Closes the "Unpack29 PPMd-mode (all current `rarvm/` fixtures are LZ-mode)"
gap from `doc/IMPLEMENTATION_GAPS.md`. The `rarvm/` archives all use LZ
blocks because the `-m5` flag alone selects LZ for binary-ish input; PPMd
requires `-mct` (method-by-context-text) to be biased on a text payload
that compresses better through PPMd than LZ.

## Files

| File | Notes |
|---|---|
| `ppmd_lorem_rar300.rar` | RAR 3.00 archive of 127 KB deterministic lorem ipsum text. Method byte `m5b` (vs `m5a` for LZ — `b` indicates the PPMd block path). UnpVer = 29. Exercises the LZ↔PPMd block transition mechanics described in `RAR15_40_FORMAT_SPECIFICATION.md` §19.2. |
| `ppmd_escape_rar300.rar` | RAR 3.00 archive of a 64 KB text-like payload containing repeated literal `0x02` bytes. The default RAR PPMd escape character is `0x02`, so this pins the literal-escape path where an encoded byte equals the active escape character. |
| `ppmd_mixed_rar300.rar` | RAR 3.00 archive containing the 127 KB lorem text followed by 64 KB deterministic binary data, generated with the same `-m5 -mct` switches. This keeps coverage on multi-file PPMd/LZ method selection without solid state. |
| `ppmd_solid_rar300.rar` | RAR 3.00 solid archive with two text members. This is a known failing `rars` fixture for solid PPMd model reuse; it is kept to pin the remaining gap. |

`expected/ppmd_lorem_rar300.rar.lt.txt` is the verbatim `Rar.exe lt` output.

## Reproduction

```sh
python3 scripts/generate-ppmd-fixture.py
```

Source files are regenerated deterministically by the script:

- `sources/lorem_127k.txt`: seeded `random.Random(0)` word picker over a
  24-word vocabulary, capped at exactly 127 KiB.
- `sources/escape_64k.bin`: repeated text phrase containing literal `0x02`.
- `sources/binary_64k.bin`: seeded `random.Random(29)` bytes.
- `sources/solid_lorem_a.txt` / `sources/solid_lorem_b.txt`: seeded lorem
  variants for solid-state reuse testing.

## What this fixture exercises in the spec

- §19.1 PPMd Block Parameters: the parameter byte (max model order, reset,
  escape-char flag), the conditional `MaxMB` and `EscChar` follow-up bytes,
  and the 4-byte range coder init.
- §19.2 PPMd / LZ transition mechanics: byte-aligned mode switch via
  `ReadTables30`, GetChar-based reads.
- §19.4 PPMd algorithm reference: the actual range-coder + model context
  exercise.
- `PPMD_ALGORITHM_SPECIFICATION.md` §13.2 escape-value handling: the lorem
  payload is large enough to trigger model rescaling (§11) and SEE updates
  (§9), and likely enough literal escape-byte occurrences (default `EscChar
  = 2`) to exercise the `NextCh == 1` "literal byte equal to escape value"
  path.
