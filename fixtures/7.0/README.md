# RAR 7 (WinRAR 7.21 beta 1) Fixtures

Generated by `scripts/generate-rar7-fixtures.py` using WinRAR 7.21 beta 1
(x64) installed at `_refs/wineprefixes/winrar721/`.

## What "RAR 7" actually means

The "RAR 7" name covers two distinct things, only one of which produces
new wire format observable from typical inputs:

1. **Unpack70** — the extended-distance LZ codec with `DCX = 80` distance
   slots (vs Unpack50's `DCB = 64`), supporting dictionaries up to 1 TiB.
   This **only engages** when the on-disk dictionary exceeds 4 GiB, which
   requires ≥4 GiB of input data. The CompInfo `Algorithm version` bit is
   set to 1 only in this case. Not generated here — see
   `IMPLEMENTATION_GAPS.md`.

2. **New RAR 7-only switches** that emit new wire-format features at any
   input size:
   - `-ams` (archive name + time save) — embeds the archive's own filename
     and creation time in the main header as a new extra record.
   - `-om[=lst]` (Mark of the Web propagation) — copies an NTFS
     `Zone.Identifier` ADS from the source file into the archive. Not
     generatable from wine on Linux because the source file would need a
     real NTFS ADS attached.
   - `-me<par>` (encryption parameters) — finer-grained KDF/cipher control.
     Default behaviour is unchanged from RAR 6.x.

## Bit-identical default output vs RAR 6.02

WinRAR 7.21's default `Rar.exe a -m3 …` output is **byte-for-byte identical**
to WinRAR 6.02's for the same input (modulo timestamps). Verified by
comparing the first 64 bytes of `r7_default.rar` and `r6_default.rar`
generated from the same `hello7.txt` source. Both produce
`Rar!\x1a\x07\x01` (RAR 5.0 wire format).

This means: a reader that handles `fixtures/5.0/` correctly will handle the
default RAR 7.21 output without changes.

## Coverage gaps not generated

- **Unpack70** (`DCX = 80` extended distance alphabet, dictionary > 4 GiB):
  requires ≥4 GiB of input data so the encoder doesn't clamp the dict.
  Not gated on tooling — purely a disk-space / fixture-size constraint.
- **MOTW propagation** (`-om`): wine on Linux doesn't preserve NTFS
  alternate data streams on the source file; needs a real Windows host or
  a hand-attached `Zone.Identifier` stream via custom tooling.
- **`-me` encryption parameter combinations**: the defaults match RAR 6.02
  so a separate fixture wouldn't differ on the wire. Worth adding once the
  spec text covers what each `-me` parameter actually changes in CompInfo.

## Reproduction

The generator script lives at `scripts/generate-rar7-fixtures.py`. It runs
the `Rar.exe` invocation in a wine subshell against this prefix. The same
script can be extended with new fixtures as more RAR 7 features become
testable.
